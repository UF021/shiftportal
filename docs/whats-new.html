🔒 Security Design

  • ✓ Generic error message — never reveals if an email exists (anti-enumeration)
  • ✓ Token is UUID4 (128-bit entropy), impossible to guess
  • ✓ Token destroyed immediately after use — cannot be replayed
  • ✓ 15-minute expiry — expired tokens auto-rejected
  • ✓ Password hashed with bcrypt before storage
  • ✓ Org-scoped: can only reset within the correct organisation
  • ✓ Only active accounts can reset (inactive staff blocked)

🎛️ UX Design

  • ✓ 3-step wizard — clear progress, no confusion
  • ✓ Show/hide toggles on both password fields independently
  • ✓ Live mismatch warning before the user submits
  • ✓ Branded per organisation — correct logo and colour throughout
  • ✓ Slug-aware routing: /forgot-password/ikan-fm or any org slug
  • ✓ Success screen with direct "Sign In →" button
  • ✓ "Back to Sign In" escape hatch on step 1

Feature 2
🕐 Clock-Out Time in HR Manual Shift Entry
HR managers can now record a complete shift — both clock-in and clock-out — in a single manual entry. Before this, HR could only create a clock-in and had to wait for a separate clock-out entry. Useful for bulk imports, scanner outages, or correcting missed events.

🌙 Overnight Auto-Detection

  • ✓ If clock-out time ≤ clock-in time, the backend auto-adds 24h — no manual checkbox needed for standard overnight shifts
  • ✓ "Overnight shift" checkbox forces the +24h even when times appear valid (e.g. 22:00 in → 06:00 out)
  • ✓ Duration preview in the UI reflects overnight calculation in real time
  • ✓ Same logic as QR clock-out — consistent everywhere

🎛️ Dynamic UI Behaviour

  • ✓ Clock-out field is optional — leaving it blank creates a clock-in only (old behaviour)
  • ✓ Overnight checkbox only appears once a clock-out time is entered
  • ✓ Duration preview only shows when both times are filled
  • ✓ Button label and info note update dynamically: "Create Full Shift Entry" vs "Create Clock-In Entry"
  • ✓ After submit, clock-out and overnight fields reset to blank
API Reference
Endpoints Changed or Added
All endpoints are tenant-scoped. Auth endpoints are public; clock endpoints require HR JWT.
Method Endpoint Description Status

GET /auth/org/:slug Fetch org branding — used by ForgotPasswordPage (existing) Existing

✅ No Email Enumeration

Whether the email exists or the DOB is wrong, the same error is returned: "We could not verify your identity." An attacker cannot probe for valid email addresses.

⚠ Token is Single-Use

The UUID4 reset token is deleted from the in-memory store the moment it is used. If a user hits "Set New Password" twice, the second attempt fails — no replay possible.

🔵 Org-Isolated Identity

The /verify-identity call requires org_slug. A valid email+DOB at Organisation A cannot be used to reset a password at Organisation B.
